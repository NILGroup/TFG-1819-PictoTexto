\section{Conclusiones}
%-------------------------------------------------------------------
\label{cap7:sec:Conclusiones}
En la actualidad la comunicación es un elemento indispensable para cualquier ser humano, ninguno de nosotros sería capaz de imaginarse su vida sin poder comunicarse con otras personas. Sin embargo, en nuestra sociedad hay personas con ciertas diversidades funcionales que les impiden poder tener una comunicación efectiva, por más simple que pudiera llegar a ser lo que quieren decir.
La aparición de diferentes alternativas de comunicación, como pueden ser los pictogramas, han ayudado a estas personas a mejorar su comunicación. A pesar de ello la comunicación con pictogramas continua siendo una comunicación muchas veces reducida al entorno más cercano o a personas que poseen el mismo tipo de diversidad. Esto es debido a que en la actualidad la comunicación con pictogramas no es universal ni comprensible para cualquier persona.
Por todo ello, vimos necesario desarrollar una herramienta que facilitase la comunicación de las personas que necesitan los pictogramas para comunicarse con otras personas que no comprenden los pictogramas. \\  
El objetivo principal en este TFG era desarrollar una aplicación web que permitiese traducir frases escritas con pictogramas a lenguaje natural con el fin de ayudar a aquellas personas que necesitan los pictogramas para comunicarse. Para este objetivo principal se ha conseguido realizar una traducción de frases principalmente simples por lo que la cobertura de las posibles traducciones no es del 100\%, además hace falta añadir más alternativas para el reconocimiento de las frases con pictogramas ya que ahora mismo se dispone de un buscador de pictogramas mediante palabra e impedirá que algunos usuarios puedan comunicarse de esta manera.\\ 
Otro punto muy importante que hemos intentado realizar es que dicha aplicación fuera lo más cómoda, intuitiva y accesible posible a fin de poder llegar a la mayor cantidad de personas. Aunque se ha conseguido realizar una aplicación muy intuitiva y accesible, si es verdad que ahora mismo el tener que usar un buscador de palabras para encontrar los pictogramas limita el número de personas que pueden usar la aplicación, una vez se implementen nuevas formas de traducción pensamos que la aplicación será muy fácil de usar.\\ 
No hay que olvidar que en la actualidad no hemos encontrado otra aplicación que realice traducciones de pictogramas a texto, por tanto aunque no se ha conseguido desarrollarla al 100\%, si que podemos decir que se han asentado unas bases sólidas sobre las que seguir desarrollando este trabajo.\\   
Otro objetivo de este trabajo era usar servicios web totalmente independientes unos de otros y que además se pudieran integrar dentro de otras aplicaciones. Lo cual se ha conseguido realizar, ya que ahora mismo cualquiera de nuestros servicios web puede ser utilizado usado por otro programador garantizando su funcionamiento correctamente.
Además hemos conseguido crear una parte Front-End basada en componentes que son también reutilizables en otras aplicaciones. Nuestro buscador de pictogramas puede ser utilizado para cualquier aplicación relacionada con el uso de pictogramas, ya que dicho buscador devuelve todo los pictogramas relacionado con una palabra.
Otro componente que puede ser reutilizable, es nuestro traductor de pictogramas a lenguaje natural donde el usuario podrá poner una secuencia de pictogramas y mover dichos pictogramas o borrarlos a su antojo.         

Este proyecto además nos ha permitido aplicar muchos de los conocimientos adquiridos en las asignaturas cursadas durante el grado a un proyecto grande y con una utilidad más allá del ámbito académico. Entre las asignaturas que más nos han ayudado en este TFG cabe destacar las siguientes:
\begin{itemize}
    \item \textbf{Ingeniería del Software}, \textbf{Modelado del Software} y \textbf{Gestión de Proyectos Software}. Donde nos enseñaron a tener la capacidad de gestionar proyectos software de una manera correcta y eficaz. Adoptando algunas de las características propias de las metodologías ágiles, además de una metodología adaptativa a las necesidades del trabajo.
    \item \textbf{Tecnología de la programación} y \textbf{Gestión de Información en la Web}. Donde nos enseñaron dos de los principales lenguajes utilizados en este TFG: Java y Python.
    \item \textbf{Aplicaciones Web}. En esta asignatura nos enseñaron los conocimientos sobre HTML, CSS y Javascript usados en el desarrollo de nuestra aplicación web.
    \item \textbf{Prácticas en empresa}. En la empresa donde se realizaron las practicas se utilizaba la herramienta de Angular, por lo que hemos decidido integrarla al proyecto para la parte Front-End.   
    \item \textbf{Ética, Legislación y Profesión}. En esta asignatura nos informaron sobre el uso de las licencias, con las que proteger nuestro trabajo así como aprender a utilizar de una manera correcta el software libre en nuestro trabajo y así respetar la licencia de la API de Arasaac. 
\end{itemize}

Además este TFG también nos ha aportado muchos conocimientos nuevos, entre los que destacan: configuración de un servidor virtual y el uso de Latex.

\section{Trabajo Futuro}
%-------------------------------------------------------------------
\label{cap7:sec:Trabajo Futuro}
Con el objetivo de mejorar y completar algunas funcionalidades de cara a obtener una aplicación más completa y que ofrezca una mayor cobertura, pensamos que se podrían marcar como trabajo de futuro la realización de lo siguiente:

\begin{itemize}
    \item \textbf{Reconocimiento de pictogramas}. Implementar otras alternativas en la aplicación aparte del buscador de pictogramas, para crear mensajes con pictogramas. Una nueva alternativa podría ser: la posibilidad de cargar directamente en nuestra aplicación un mensaje con pictogramas.
    \item \textbf{Evaluación de la usabilidad}. Realizar un diagnostico sobre si la aplicación creada es simple, intuitiva y amigable con el usuario, para ello se podría llevar dicha aplicación a personas que utilizan los pictogramas y analizar que les gusta, que no y como de usable ven nuestra aplicación.
    \item \textbf{Mejora en la lectura de los pictogramas}. Estudiar la posibilidad de cargar las imágenes de los pictogramas de una manera alternativa sin hacer peticiones a la API de Arasaac, como pudiera ser mediante el uso de una base de datos que posea la aplicación para así proporcionar más velocidad cuando se busquen los pictogramas.
    \item \textbf{Incremento en la complejidad de las frases a traducir}. Actualmente nuestra aplicación realiza un estudio correcto de las frases simples, aquellas que son un sustantivo con un determinante como sujeto, un verbo y algunos complementos en el predicado. Se podría implementar que el sujeto esté formado por varios sustantivos, así como poder realizar traducciones de más de un verbo, en el predicado mejorar las traducciones con preposiciones y estudiar la posibilidad de añadir más complementos.
\end{itemize}

El factor principal por el que no se han podido realizar las funcionalidades comentadas no ha sido otro que la falta de tiempo. Nos gustaría en un futuro poder seguir mejorando nuestra aplicación con el fin de completarla lo máximo posible y así poder ayudar a las personas que lo necesiten.
