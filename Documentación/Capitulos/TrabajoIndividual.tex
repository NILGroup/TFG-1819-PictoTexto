\section{Trabajo Individual de Jose Maria}
%-------------------------------------------------------------------
\label{cap6:sec:Jose Maria}

Lo primero que realice fue un estudio  de aquellas tareas del estado del arte, de las cuales estábamos seguros que tendríamos que tratar para comenzar con la redacción de este documento. La investigación inicial se centro en en el estudio de los SAACS, poniendo especial atención en los pictogramas y los sistemas pictograficos, y de manera general Servicios Web. 

En la búsqueda de información sobre los SAACS, analice los sistemas pictograficos más extendidos y la comunicación a través de  pictogramas. Tras esta investigación decidimos que usaríamos los pictogramas de ARASAAC para el desarrollo de este Trabajo de Fin de Grado.

Uno de los requisitos indispensables de este proyecto, era la necesidad de incluir la aplicación dentro de un Servicio Web, por lo cual investigue sobre cuales eran las diferentes opciones para crear un Servicio Web en Python, y dada su potencia, facilidad de uso y la gran comunidad decidimos utilizar Django. Una vez expuesto y estudiado su uso, desarrolle una prototipo tecnológico, para probar su utilidad y como primer acercamiento a la tecnología.

Dado que el proyecto era muy amplio decidimos, crear una metodología de desarrollo para que facilitar el trabajo. Por lo cual realice un estudio de las diferentes metodologías de desarrollo software, y después plantee en una reunión tanto una metodologia como el tablero que usamos a lo largo del proyecto. Después documente la metodología en la memoría en el capitulo \ref{cap4:MetodologiaDeTrabajo} y cree el tablero en GitHub \footnote{https://github.com/NILGroup/TFG-1819-PictoTexto/projects/1}.

El siguiente paso necesario, era un estudio sobre los diferentes sistemas de generación de lenguaje natural parte fundamental del proyecto. Investigue y documente la base teórica de la generación de lenguaje natural, una vez termine esto pase a la búsqueda de diferentes frameworks de generación de lenguaje natural en español, tras la cual encontré el framework SimpleNLG-ES, que es el unico  framework de generación de lenguaje natural en español que encontré. Una vez presentados los resultados desarrolle un prototipo tecnológico utilizando dicho framework. Este prototipo muy simple permitía enviarle un sujeto, verbo y un complemento directo. Dado que SimpleNLG-ES es un framework Java, y la base de los microservicios web están implementados en Python, decidi encapsular el framework en un servicio web al cual se llamaba desde el servicio web de Django.
Para generar este servicio web, analice diferentes vías de servicios web Java, y realice dos prototipos uno con WildFly\footnote{http://www.wildfly.org/} y otro con Apache Tomcat\footnote{http://tomcat.apache.org/}, y teniendo en cuenta diferente factores como rendimiento o la comunidad decidí implementar Tomcat. Durante el desarrollo de este prototipo surgió el problema de recibir datos en formatos JSON en un servicio web Java, tras analizar las diferentes posibilidades que encontré decidí optar por usar el framework GSON.

Con el aumento de los servicios web implementados, decidí refactorizar el código y diseñe una arquitectura que fomentara la rápida integración de nuevos servicios, que además sean completamente independientes entre ellos, y ayudaran al mantenimiento de la aplicación, una vez investigado y definida la arquitectura. Una vez termine el trabajo de implementación, documente los servicios web, los principios de la arquitectura, en el capitulo \ref{cap2:sec:ServiciosWeb}.

Dado el limite del framework Django para la gestión de aplicaciones web desde el lado del cliente, desarrolle una aplicación web fron-end en Angular CLI, que después integre con la aplicación Django, para de esta manera obtener mayor funcionalidad, flexibilidad y descargar parte del trabajo en el cliente. Una vez terminada la implementación documente la aplicación. 

Durante el desarrollo de la aplicación nos encontramos con un problema con la detección de palabras de Simple-NLG, asi que decimos delegar esa tarea en un servicio web dentro de la aplicación Django, que hiciera uso del framework Spacy. Después de investigar su uso, desarrolle el servicio web de detección de palabras y posteriormente documente su funcionamiento.  Además de este servicio decidimos que era necesario, crear un servicio web que detectara el tiempo verbal en el que se debería conjugar la frase. Con la respuesta de este servicio se llamaría a un servicio web u otro dentro del servicio de generación de lenguaje natural. Una vez estudiados los pasos a seguir desarrolle los tres servicios web. Para los servicios de generación de lenguaje tuve que hacer una refactorización el servicio ya creado, para abstraer el comportamiento común de los diferentes servicios web de la generación de lenguaje. Una  vez terminado todo los descrito documente los servicios web.
